Purplle Retail Intelligence

A state-of-the-art Computer Vision & Spatial Analytics platform designed to transform raw CCTV footage into actionable operational intelligence for physical retail stores.

🚀 Quick Guide

Designed for both technical operators and store executives, getting started takes only a few steps.

For Non-Tech Users

1. Dashboard View: Access the main UI. You are in Executive Mode by default, seeing high-level metrics (Footfall, Dwell Time, Heatmaps).

2. View Heatmaps: Click the Heatmap tab to see where customers spend the most time.

3. Live Feed: Watch the Event Feed on the right to see customers entering, dwelling, and queueing in real-time.

For Tech Experts

1. Run API: ./run-api.sh starts the FastAPI backend on port 8000.

2. Run Pipeline: Execute python pipeline/detect.py to process raw MP4 files. This parses YOLO detections into JSON events.

3. Engineer Mode: Toggle "Engineer Mode" in the UI to view raw bounding boxes, raw event JSON feeds, and zone overlays.

🎯 The Problem It Solves

Physical retail suffers from a massive data deficit compared to e-commerce. While online stores track every click, hover, and bounce, physical stores are a "black box" once a customer walks through the door.

  • Layout Inefficiency: Retailers don't know which shelves attract attention but don't convert to sales.
  • Staff Allocation: Managers struggle to match staff schedules with real-time customer influx.
  • Queue Abandonment: Long lines at checkout lead to lost revenue, but managers are only aware after the queue has formed.
  • Manual Auditing: Relying on human security guards to manually watch CCTV arrays is expensive, error-prone, and unscalable.

⚙️ System Architecture & Portability

The system is built as a decoupled, asynchronous, and highly portable platform.

1. Extreme Portability

The entire application is designed to run on the Edge (e.g., a localized store server or NVIDIA Jetson) or in the Cloud.

  • Dependency Isolation: Managed via strict requirements.txt with pinned versions.
  • OS Agnostic: Runs seamlessly on macOS (M-series CoreML), Linux (CUDA), and Windows. Path handling uses robust os.path standard libraries, bypassing OS-specific slash issues.
  • Zero-Config Database: Uses SQLite for portability. You do not need to stand up a complex PostgreSQL cluster to deploy a store. A single `.db` file holds all analytical metrics, making backups as simple as a file copy.

2. Scalability

Can this handle a busy store with thousands of events?

  • Asynchronous Backend: The FastAPI backend uses async/await extensively. It can ingest thousands of JSON events per second from the CV pipeline without blocking UI requests.
  • Batch Processing: The CV pipeline does not hit the database for every single frame. It buffers tracking states in memory and performs bulk inserts, drastically reducing I/O overhead.
  • Adaptive Inference: The pipeline dynamically scales image resolution and skip-frames based on hardware capability (CPU vs MPS vs CUDA).

🛡️ Reliability & Security

Enterprise software must be resilient to failures and secure against intrusions.

  • WAL Mode: The SQLite database is configured with Write-Ahead Logging (WAL). This ensures that heavy read operations (the Dashboard polling) do not block heavy write operations (the Pipeline ingesting), preventing database locking errors.
  • Robust Error Handling: The Pos Loader and Event Ingestion engines wrap operations in transactional blocks. If a corrupt CSV or malformed JSON is uploaded, the transaction rolls back, preventing database corruption.
  • Security Hardened:
    • XSS Prevention: The UI employs strict DOM sanitization (escapeHtml) before rendering any user-sourced data (like Camera IDs or Event Labels) into the DOM.
    • Path Traversal Protection: API endpoints strictly validate video file paths against a secure base directory, preventing malicious actors from accessing system files like `/etc/passwd`.
    • SQL Injection: All database queries utilize SQLAlchemy's ORM or parameterized execution. Zero string-concatenation SQL queries exist in the codebase.

📷 Multi-Device Configuration

Real stores do not have just one camera. They have arrays of cameras covering different zones.

  • Zone Mapping: The application maps raw (x,y) pixel coordinates from multiple cameras into a unified store layout.
  • Camera Agnostic: The pipeline accepts an arbitrary number of RTSP streams or MP4 files. Each camera registers its events tagged with a unique camera_id.
  • Global Unique Identifiers: Person Re-Identification (ReID) ensures that if "Visitor 42" leaves Camera A's FOV and enters Camera B's FOV, they are tracked as the same session, rather than generating duplicate footfall counts.
